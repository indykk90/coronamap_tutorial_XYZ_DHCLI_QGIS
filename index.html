HERE XYZ Studio - interactive, visual, web-based application for accessing XYZ Hub Data and creating maps within minutes.

HERE Data Hub CLI - command line interface which uses an interactive command window to enter instructions.

XYZ Token Manager - lets you manage the access tokens for your spaces.

XYZ Hub Connector - official HERE QGIS plugin which allows to connect data between XYZ Studio and QGIS.

QGIS - free and open-source cross-platform desktop geographic information system (GIS) application that supports viewing, editing, and analysis of geospatial data.

GIS (Geographic Information System ) - system designed to capture, store, manipulate, analyze, manage, and present spatial or geographic data.

Shapefile/ ESRI Shapefile/ SHP - geospatial vector data format for geographic information system (GIS) software. The shapefile format can spatially describe vector features: points, lines, and polygons. Each item usually has attributes to describe it.

Attribute table - tabular information about shapefile.

Geographic coordinate system - enables every location on Earth to be specified by a set of numbers, letters or symbols. A common choice of coordinates is latitude, longitude and elevation. To specify a location on a plane requires a map projection.

GeoJSON - format for encoding a variety of geographic data structures, based on JSON.

JSON - open standard file format, and data interchange format, that uses human-readable text to store and transmit data objects consisting of attribute–value pairs and array data types (or any other serializable value).

CSV - delimited text file that uses a comma to separate values. Each line of the file is a data record. Each record consists of one or more fields, separated by commas. The use of the comma as a field separator is the source of the name for this file format. A CSV file typically stores tabular data (numbers and text) in plain text, in which case each line will have the same number of fields.

HTML - standard markup language for documents designed to be displayed in a web browser.

Hyperlink - reference to data that the user can follow by clicking or tapping.

Node.js - open-source, cross-platform, JavaScript runtime environment that executes JavaScript code outside of a web browser.

Command prompt/ cmd.exe - input field in a text-based user interface screen for an operating system or program.

Introduction

This training was prepared to show functionalities of the HERE XYZ Studio together with QGIS. HERE XYZ Studio is an interactive, visual, web-based application for accessing XYZ Hub Data and creating maps within minutes. This tutorial doesn't require coding skills.

Training objectives

In this tutorial, you're going to build a coronavirus map. Hence this is a trending topic right now around the globe, I will demonstrate how to use the power of XYZ Studio together with QGIS. In this tutorial, we'll create a coronavirus map for Poland.

This map will:

  • Display shapefiles (points and polygons)
  • Be updated using QGIS plugin
  • Display hyperlinks (popup windows)
  • Allow you to style your layers any way you want it

Type it in Node.js command prompt.

here configure account

Create new project

After first log in to the XYZ Studio, you won't see any projects. To create a new one, click Create new project button or click on the circle icon with + inside (which is located below Create new project button).

Add polygon shapefile to QGIS

Add downloaded shapefile to QGIS. Remove almost all fields from the attribute table, we are going to create new ones.

Note: Keep only existing field with the voivodeships names.

Attention: Don't leave created columns (fields) empty. Each column should be filled in by at least one row. XYZ Studio doesn't recognize empty fields. For this reason, fill created fields with template or random data. We will modify them later.

↑ Shapefile provided by EUROSTAT, added to the QGIS.

To display specific field (e.g. information about number of infected people) on a map in XYZ Studio, create this field in the attribute table of the polygon shapefile.

Note: Keep in mind while creating fields in the attribute table, that QGIS supports only field names with max 10 characters long.

We are going to create following fields:

  • Voi - voivodeship, type: string
  • Status - this field informs whether voivodeship is infected or not, type: string
  • Infected - number of infected people, type: integer
  • Isolation - number of people in isolation, type: integer
  • Source - data source (we will use direct hyperlinks), type: string
  • Hosp - number of hospitalized people, type: integer

↑ Attribute table, ready to be filled in (information about voivodeship names saved from the original shapefile source - EUROSTAT, rest of the fields filled by template/random data - don't leave empty fields).

Adding data using Data Hub CLI

Note: For more information about Data Hub CLI, visit: https://developer.here.com/tutorials/using-the-xyz-cli/

To add larger data to the HERE XYZ Studio, we need to use Data Hub CLI. First of all, open Node.js Command Prompt.

Next step is to upload our first Shapefile (let's use previously prepared polygon shapefile - voivodeship_poland.shp), type:

You will be asked to provide a title and description for the new space.

After this, your shapefile will be uploaded.

To upload another shapefiles (cities, hospitals etc.), follow the same rule.

Adding data to the XYZ project

To add all uploaded data to our map, please click the Data icon located below the description of your project.

You will be redirected to the My Spaces section. Select all layers you want to add to the project and click the Add dataset button located in the bottom right corner. If you select more than 1 dataset, the button will change name to Add 1 dataset, Add 2 datasets, Add 3 datasets etc.

Connecting XYZ Studio with QGIS

XYZ Token Manager allows connection between XYZ Studio and QGIS.

To connect shapefiles from the XYZ Studio with QGIS, visit website provided below:

Note: XYZ Token Manager: https://xyz.api.here.com/token-ui/index.html

Log in (use your XYZ Studio credentials) and select shapefiles uploaded in the previous step. Tick All option.

When layers are ticked, scroll your page to the very bottom and click the Generate Token button.

Provide a description for token and click the Generate Token button.

Token will be generated, click the Copy Token button and switch to QGIS.

In QGIS, click the XYZ Hub Connection button .

New window displayed. Click the Manage Token button.

Click the Add button and fill out two fields, Name and Token:

  • Name: name for your token (depends on you)
  • Token: paste a Token copied from the XYZ Token Manager

Click Ok, select relevant Token from the list and click Ok.

Click the Use button. Relevant shapefiles will display. Click on each and then click the Load button (you have to add them one by one using the Load button).

Attention: Keep Mode as a static (below the Load button)!

Congratulations! Shapefiles were added to the QGIS.

Note: As you can see, polygons were split by single polygons and multi polygons. Don't worry, all is fine.

First of all, we'll play with the polygon layer.

According to the local media, not all voivodeships have been infected by coronavirus. Let's add relevant numbers to the attribute table of the polygon layer. As a source, provide a link to the local newspaper/ website.

↑ Updated attribute table of the polygon layer.

Our polygon shapefile is ready, now it's time to edit and update our point shapefile. For now, we have only one city added, capital of Poland - Warsaw. According to the data provided by the Ministry of Health (Poland), other cities have been also infected. Let's add them to the point shapefile.

Switch off visibility of the polygon layer (voivodeship) and add background to the map.

When your map is ready to go, save your changes. Now it's time to push your changes to the XYZ Studio. To do this, select the layer you want to push (from the layers list) and click the highlighted button of cloud (Push changes). This icon is always located next to the XYZ Hub Connection button.

↑ Push changes icon.

The green bar informs that data has been pushed successfully to the XYZ Studio. Repeat this step with all modified layers.

Polygon layer

Our project is ready for styling. This is our view after pushing modified shapefiles to the project.

In this step you can change colors, symbols, outline color and thickness as you wish. You can also display labels of each layer.

Let's color voivodeships by using two colors. We'll split voivodeships by those infected and uninfected by coronavirus. Moreover, let's display each voivodeship name.

To do this, we need to add a new style group. Click on the polygon layer (voivodeships), a new view will display. Click the + Add new polygon style button.

New window will appear. From the dropdown list select:

  • Property: status
  • Condition: equals
  • Value: infected

Provide also a name for your style rule. Click Ok.

When it's ready, all infected voivodeships will change color. To modify the color, click on the relevant style group.

A little task for you: Repeat this step and create a group for the uninfected voivodeships. :-)

The result (voivodeships split by infected and uninfected):

The result:

Now it's time to publish our map. You can easily share a link to your map or copy & paste a snippet of code that you can directly include on a web page or in a blog.

By default, every map you create with HERE XYZ Studio is private and visible only to you until you decide to publish it.

All you have to do to publish the map is to turn publishing on (you can turn it off anytime again as well if you wish). This button is located in the bottom left corner of your screen.

When the map is published, you also see the URL where it can be reached. You can copy this and share it with people directly or you can link to it from somewhere like a blog or wiki page.

You can also copy the EMBED code and use it directly in the code of your website or blog. It's a bit of HTML that will show your map directly on the page.
